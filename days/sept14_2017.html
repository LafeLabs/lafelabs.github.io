Getting Started with Scripts

A script is a specific type of computer program. "Script" generally refers to the type of computer program which a human can read. This is in contrast to "compiled" code, which is code that has been translated from human-readable code to pure binary which only the machine can read. Historically, compiled code was important because it was so much faster. Today, as computers have gotten exponentially faster, the purpose of compiled code is often for closed-source programmers to keep their work secret, and to prevent users from exercising their fundamental human rights to have power over their own hardware. Behind almost every piece of compiled code there exists a crime against knowledge and freedom.

There are numerous languages which can be considered scripting languages, as well as languages with can be either compiled or scripted. JavaScript, as we'll be using it here, consists of scripts embedded in the html document. This thus completes the "Holy Trinity" of the Open Web: HTML, CSS, and JavaScript. Just as we embed CSS into HTML with the "style" tag and can use it to control how things look at any time in making HTML documents, the "script" tag will let us do the same thing with actions. JavaScript is how you make a page actually do something, rather than be just a static piece of information. There are numerous other languages you'll hear people saying they used to write a web application, but those will generally run on some remote server, such as used for Facebook or Google. Not so JavaScript. A script that lives inside your html document is run on YOUR computer. If you don't use remote libraries or scripts, this can run without any connection to the Internet at all! A physically isolated machine with a web browser can run the code, making it possible to, with cheap modern devices like Android burner phones, build all kinds of automation and robotics technology with nothing but html that runs in a browser! This is the true power of Geometron and the longer term goal of this work: a truly free and decentarlized method of industrial production. And this is why it's so important to focus on JavaScript over its varous cousins like Java, C++ or Python. Those languages are great for building commercial products and large web sites for millions of users, built by professionals, but to build free robots that can be deployed on the Street by amateurs, nothing can beat raw HTML for its freeness and universality.

It's worth going into some more detail about the way we will use software in the Geometron system of knowledge and how it differs from that of the professional software developer. In general, software development is an expensive endeavor, requireing that large amounts of central bank debt currency come from somewhere to compensate all the professioanls working on it. To support the cost of labor, revenue must be at least one software person's salary, and since each user who clicks on a given web site is likely to only deliver a small amount of revenue, this means the only projects which can pay for the labor required to build something interesting are those with very large numbers of users. To put real numbers on it, suppose your team of 10 software people cost 5,000 dollars a month each. That's 50 grand a month. Now if you get one percent of your users to spend 100 dollars(as might be expected for some specialty e-commerce site) or 1 dollar per click as might be the case for a referral system of reveneue, you need 50,000 clicks a month at a dollar a click to keep those software people paid. This is more than 1000 people a day who need to be going to your site every day in order to just barely stay afloat. But under the system of investor-funded computer capitalism, that math is woefully wrong, and it's much worse than this. In order to initially get the 50k/month to set this up, you need investors, and they all invest in mostly idiotic companies which will mostly fail, so the few that succeed all need to make them massive returns of 10x or 20x or more of their initial investment. This means that getting to the keeping-everyone-paid level will definitely result in them losing money. Therefore the company must be structured so that it has sustained exponential growth or it will fail due to how investor deals are all structured(they do this to put pressure on founders to constantly push for exponential growth and since they regard companies that simply break even as failures, which they call the "walking dead"). Thus that group of 10 programmers with a thousand hits a day need to be desparately working to turn it into 100,000 hits a day or they'll all be unemployed in 18 months. Any tool that makes that happen faster and with lower chance of failure will therefore be used, no matter what. The stakes are high, and the pace is fast, the scale global.
